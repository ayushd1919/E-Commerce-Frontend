auth.service.ts auth

  • login(email, password) → POST /api/auth/login
  • register(name, email, password) → POST /api/auth/register
  • logout() → POST /api/auth/logout
  • forgotPassword(email) → POST /api/auth/forgot-password
  • verifyCode(email, code) → POST /api/auth/verify-code
  • resetPassword(email, code, newPass)
  • currentUser$ BehaviorSubject — shared state
  • isLoggedIn(), getRole() helpers

product.service.ts http

  • getProducts(filters, page) → GET /api/products
  • getProductById(id) → GET /api/products/:id
  • search(keyword, filters, page) → GET /api/products/search
  • getByTaxonomy(type?, cat?, subcat?)
  • createProduct(data, image?) multipart POST
  • updateProduct(id, data, image?) multipart PUT
  • deleteProduct(id) → DELETE /api/products/:id

cart.service.ts state + http

  • cart$ BehaviorSubject — local reactive state
  • loadCart() → GET /api/cart (on login)
  • addItem(productId, qty) → POST /api/cart
  • updateQty(productId, qty) → PUT /api/cart/:id
  • removeItem(productId) → DELETE /api/cart/:id
  • clearCart() called after order placed
  • getTotal() computed from cart$

order.service.ts http

  • placeOrder(cartItems, paymentMethod) → POST /api/orders
  • getMyOrders() → GET /api/orders/my
  • getOrderById(id) → GET /api/orders/:id
  • getAllOrders() → GET /api/admin/orders (admin)
  • getAdminOrderById(id)

user.service.ts http

  • updateProfile(name, email) → PUT /api/users/me
  • changePassword(current, newPass)
  • getAllCustomers() → GET /api/admin/customers
  • lockAccount(userId) → PATCH /api/admin/customers/:id/lock
  • unlockAccount(userId) → PATCH /api/admin/customers/:id/unlock

taxonomy.service.ts http

  • getTypes() → GET /api/taxonomy/types
  • getCategories(typeId)
  • getSubCategories(categoryId)
  • Used in product filter panel and admin product form for cascading dropdowns

auth.guard.ts CanActivate

  • Checks auth.service.isLoggedIn()
  • If not logged in → redirect to /auth/login
  • Applied to: /cart, /checkout, /account/**
  • Saves attempted URL in session for redirect after login

admin.guard.ts CanActivate

  • Checks auth.service.getRole() === 'admin'
  • If not admin → redirect to / (home)
  • Applied to: entire /admin/** lazy module
  • Never trust role from localStorage — read from session service only

guest.guard.ts CanActivate

  • Prevents logged-in users from accessing /auth/login and /auth/register
  • If already logged in → redirect to /
  • Avoids double-login edge case

admin-module.guard.ts CanLoad

  • Prevents the admin bundle from even being downloaded by non-admins
  • Use CanLoad (not just CanActivate) on the lazy route
  • Complements admin.guard.ts at the route level
How guards are applied to routes
/auth/login→guest.guard
/cart→auth.guard
/checkout→auth.guard
/account/**→auth.guard
/admin/**→admin.guard+admin-module.guard (CanLoad)

credentials.interceptor.ts HTTP

  • Adds withCredentials: true to every outgoing request
  • Required for the HTTP-only cookie to be sent automatically
  • Applied globally in AppModule providers
  • No manual token header needed — cookie does the work

error.interceptor.ts HTTP

  • Catches 401 Unauthorized → clears local auth state, redirects to login
  • Catches 403 Forbidden → shows "access denied" message
  • Handles account-locked response: clears session, shows lock message
  • Catches 500 errors → shows generic "something went wrong", never raw backend error
  • Catches network errors (offline)

loading.interceptor.ts UX

  • Tracks in-flight requests with a counter
  • Emits to a LoadingService BehaviorSubject
  • Navbar spinner subscribes to LoadingService.isLoading$
  • Properly decrements even on error (use finalize())
Registration order in AppModule (order matters)
credentials.interceptor → then → loading.interceptor → then → error.interceptor
Shared (used everywhere)

navbar.component

  • Shows login/register links for guests
  • Shows cart icon + item count badge for customers (subscribes to cart$)
  • Shows "Admin Panel" link for admins
  • Logout button calls auth.service.logout()
  • Taxonomy dropdown for browsing by Type → Category

product-card.component

  • Displays product image (or placeholder), name, price
  • Clicking navigates to /products/:id
  • Used on homepage, product list, search results
  • Input: @Input() product: Product

pagination.component

  • Input: currentPage, totalPages
  • Output: pageChange EventEmitter
  • Used on product-list and search-results

filter-panel.component

  • Dropdowns for Type / Category / Sub-Category (cascading via taxonomy.service)
  • Price range min/max inputs
  • In-stock toggle
  • Output: filterChange EventEmitter with filter object
  • Used inside product-list and search-results
Feature — Auth

login.component

  • Reactive form: email + password
  • Calls auth.service.login()
  • On success: redirect to saved URL or home
  • Shows locked-account error from API

register.component

  • Reactive form: name, email, password, confirm-password
  • Frontend validation: password match, min length
  • Calls auth.service.register()

forgot-password.component

  • Step 1: enter email → calls forgotPassword → displays numeric code on screen
  • Step 2: enter code → calls verifyCode → shows new password form
  • Step 3: submit new password
  • Shows error if code expired
Feature — Products

product-list.component

  • Loads products via product.service.getProducts() or getByTaxonomy()
  • Reads type/category/subcat from route params
  • Integrates filter-panel + pagination components
  • Renders grid of product-card components

product-detail.component

  • Loads product by :id from route
  • Shows full taxonomy breadcrumb path
  • Shows image or default placeholder
  • "Add to Cart" button — visible only to logged-in customers
  • Share button: copies window.location.href to clipboard
  • Deep-linkable URL works on any device

search-results.component

  • Reads q query param from URL
  • Calls product.service.search(keyword, filters, page)
  • Cross-taxonomy results displayed in one grid
  • Integrates filter-panel + pagination
Feature — Cart & Checkout

cart.component

  • Subscribes to cart.service.cart$
  • Quantity +/− buttons → calls updateQty()
  • Remove button → calls removeItem()
  • Shows running total
  • "Proceed to Checkout" button

checkout.component

  • Shows order summary from cart
  • Payment method radio: Credit Card, Debit Card, Cash on Delivery, Bank Transfer
  • Confirm button → calls order.service.placeOrder()
  • On success: clears cart, navigates to /checkout/confirmation/:orderId

order-confirmation.component

  • Loads order by ID from route param
  • Shows items, quantities, total, payment method
  • Shown once after successful checkout
Feature — Account (customer)

profile.component

  • Reactive form pre-filled with current name + email
  • Calls user.service.updateProfile()
  • Updates auth.service.currentUser$ on success

change-password.component

  • Fields: current password, new password, confirm
  • Calls user.service.changePassword()
  • Shows API error if current password wrong

order-history.component

  • Calls order.service.getMyOrders()
  • Lists orders with date, total, payment method, status
  • Each row links to /account/orders/:id

order-detail.component

  • Loads order by ID (customer's own orders only)
  • Shows snapshot price at time of purchase, not current price
  • Full item breakdown + payment method
Feature — Admin

admin-product-form.component

  • Used for both create and edit
  • Cascading dropdowns: Type → Category → Sub-Category via taxonomy.service
  • Image file input → sends as FormData multipart
  • Preview of selected image before upload
  • Populates existing values when editing

admin-customer-list.component

  • Calls user.service.getAllCustomers()
  • Lock/Unlock toggle button per row
  • Calls lockAccount() or unlockAccount()
  • Lock takes effect immediately on next customer request

admin-order-list.component

  • Calls order.service.getAllOrders()
  • All customers' orders in one table
  • Sortable by date, customer, total
  • Links to admin order detail view

  • name: string
  • description: string

  • interface Category { id, name, typeId }
  • interface SubCategory { id, name, categoryId }

cart.model.ts

  • interface Cart { items: CartItem[], total: number }

  • items: OrderItem[]

  • interface ApiResponse<T> { data: T, message?: string }
  • interface PaginatedResponse<T> { data: T[], page, totalPages, total }
  • Used as generic wrapper for all HTTP responses
